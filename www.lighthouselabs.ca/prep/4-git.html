IV. Git & Github

Introduction

Every software project is managed in "Source Control". Source control helps developers keep track of their work and have multiple people easily contribute to the same project.

Git is by far the most popular open source option for source control, especially in the web development world. There are many alternatives to Git, but focusing on git alone is fine.

Github is a web application that hosts our Git repositories for us and makes Git even more social and collaborative. It's super cool. In fact, many open-source projects, including Ruby and Ruby on Rails are hosted there. We'll be using a bunch of these during our course.

If some of it is confusing or daunting, don’t worry. We'll be using Git and Github throughout the course so you'll get a knack for it over time.

What You'll Need

  1. Sublime Text 3

The Work

1. Github account

You should have already done this. In case you haven't yet, please create an account on Github.

2. Download and Setup Git

Step1: Download and setup git on your machine:

https://help.github.com/articles/set-up-git

Important Note: Don't follow their instructions to install or use the Github app. It's very important to become comfortable with Git from the command-line/terminal.

Step2: Setup authentication with Github

Setup SSH keys to connect from your git client to your github profile by following these instructions: https://help.github.com/articles/generating-ssh-keys

3. Setup Git

Set your global configuration for Git by following the instructions on these two pages:

  1. Github: Setting Your E-mail in Git

  2. Github: Setting Your Username in Git

  3. Next we're going to generate a public/private key pair. You keep the private key on your machine and add your public key (the .pub file) to Github. This is how Github will authenticate you. Follow the steps here: Github: Generating SSH Keys (see notes below before following this)

3. Read: Git - The Simple Guide

Don’t worry about branching, merging and tagging for now. You can skip or gloss over these three intermediate topics, for now. We will focus on these together in class.

4. Exercise: CodeSchool - Try Git

At 1.18 - Branching Out they get into branching, at which point you can choose to stop.
